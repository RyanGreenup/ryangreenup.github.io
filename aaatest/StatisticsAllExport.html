Hypotheses Testing Single Population Samples

Topic 1A | Week 1 | Material Drawn From Week 1 Lecture

When comparing population means standard normal distributions are used when the population standard deviation is known and t-distributions are used when only the sample standard deviation is known.

Steps to Hypothesis Testing

  1. Hypothesis
  2. Test Statistic
  3. Rejection Region
  4. P-value
  5. Conclusion
  6. Hypothesis
    1. Describe the Problem
    2. List the Data
    3. State the null hypothesis and alternative hypothesis
  7. Test Statistic
    1. Decide upon the correct test statistic
      1. Relative to population size, known variance etc.
    2. Find the Test statistic
  8. Rejection Region
    1. Find the Rejection region
    2. State whether or not the null hypothesis is rejected
  9. P-value
    1. Find the value of \(\alpha\) required to reject the null hypothesis
  10. Conclusion
    1. Write a statement answering the initial question relative to the confidence level

Estimate Sample Size for statistical Test1

The Confidence Interval for the population mean \(\mu\) is given by

\[\overline{x}\ \pm z_{\frac{\alpha}{2}}\frac{\sigma}{\sqrt{\text{n\ }}}\]

To find \(\mu\) within B of the sample mean it must be true that

\[z_{\frac{\alpha}{2}}\frac{\sigma}{\sqrt{n}} \leq B\]

Rearranging to solve for the sample size \(n\) gives

\[n\ \geq \frac{\text{σ\ }z_{\alpha/2\ }}{B^{2}}\]

Thus the sample size of a sample that will give a confidence interval of \(\mu\) at a confidence level of \(1 - \alpha\) to within B of the sample mean is \(n\):

\[n\ \geq \frac{\text{σ\ }z_{\alpha/2\ }}{B^{2}}\]

Hypothesis Testing and Confidence Intervals

Where the population variance \(\sigma\) is known2

If a sample is from a normal population or is practically large (30 or greater) it can be modelled using a Standard Normal Distribution.

  • Hypotheses Testing
    • Hypotheses:

      Three types of Hypotheses occur:

      1. H0: µ = g then H­a: µ < g
      2. H0: µ = g then H­a: µ > g
      3. H0: µ = g then H­a: µ / g
    • Test Statistic

      \(Z = \frac{\overline{x} - µ}{\frac{\sigma}{\sqrt{n}}}\)

    • Rejection Region

      At significance level α, H0 is rejected and Ha accepted for the following statistics:

      1. H0: µ = g then H­a: µ < g
        1. H0 fails and Ha is accepted if Z < - Zα
      2. H0: µ = g then H­a: µ > g
        1. H0 fails and Ha is accepted if Z > Zα
      3. H0: µ = g then H­a: µ / g
        1. H0 fails and Ha is accepted if |Z| > |Zα/2| => Z > Zα/2 & Z< -Zα/2
    • Confidence Interval

      A (1-α) x 100% Confidence Interval for µ is given by:

      \[\bar{x} \pm Z_{\frac{\alpha}{2}}\ \text{x\ }\left( \frac{\sigma}{\sqrt{n}} \right)\]

Where \(\sigma\) is estimated by \(s\)

  • Hypotheses Testing
    • Hypotheses:

      Three types of Hypotheses occur:

      1. H0: µ = g then H­a: µ < g
      2. H0: µ = g then H­a: µ > g
      3. H0: µ = g then H­a: µ / g
    • Test Statistic

      \(t = \frac{\overline{x} - µ}{\frac{s}{\sqrt{n}}}\)

    • Rejection Region

      At significance level α, H0 is rejected and Ha accepted for the following statistics:

      1. H0: µ = g then H­a: µ < g
        1. H0 fails and Ha is accepted if t < -tα, d.f.
      2. H0: µ = g then H­a: µ > g
        1. H0 fails and Ha is accepted if t > tα, d.f.
      3. H0: µ = g then H­a: µ / g
        1. H0 fails and Ha is accepted if |t| > |tα/2, d.f.| => t > tα/2, d.f. & t < - tα/2, d.f.
  • Confidence Interval

    A (1-α) x 100% confidence interval for µ is given by:

    \[\bar{x} \pm \ t_{\alpha/2}\ x\ (\frac{s}{\sqrt{n}})\]

Hypothesis Testing and Confidence Intervals for Proportions

Hypothesis Testing

  • Data

    Let:

    \(p = \frac{x}{n}\) , where x is the number of successes in n trials in a measured sample

    \(\pi = \frac{x}{n}\) , where x is the number of successes in n trials for a population

  • Test Statistic

    \[z = \frac{p - \pi_{0}}{\sqrt{\frac{\pi_{0}\left( 1 - \pi_{0} \right)}{n}}}\]

  • Rejection Region

    HA : \(\pi < \pi_{0}\), reject H0 when \(Z < - Z_{\alpha}\)

    HA : \(\pi > \pi_{0}\), reject H0 when \(Z > Z_{\alpha}\)

    HA : \(\pi = \pi_{0}\), reject H0 when $Z\ > Z\frac{\alpha}{2}\ $or \(Z < - Z_{\frac{\alpha}{2}}\), which is same as \(\left| Z \right| > Z_{\frac{\alpha}{2}}\)

Confidence Interval

A \(\left( 1 - \alpha \right)\) Confidence Interval for \(\pi\) can be found by:

\[p\ \pm Z_{\alpha/2}\ (\sqrt{\frac{\pi_{0}\left( 1 - \pi_{0} \right)}{n}}\]

Hypothesis Testing and Confidence Intervals for Population Variance

Hypothesis Testing

  • Hypothesis

    Any of the three hypothesis may occur:

    1. H0: \(\sigma^{2}\) = g, Ha: \(\sigma^{2}\) < g
    2. H0: \(\sigma^{2}\) = g, Ha: \(\sigma^{2}\) > g
    3. H0: \(\sigma^{2}\) = g, Ha: \(\sigma^{2}\) / g
  • Test Statistic

    \[\ t = \frac{\left( n - 1 \right)s^{2}}{\sigma^{2}}\]

  • Rejection Region
    1. H0: \(\sigma^{2}\) = g, Ha: \(\sigma^{2}\) < g
      1. H0 fails for t < \(X_{\alpha}\)
    2. H0: \(\sigma^{2}\) = g, Ha: \(\sigma^{2}\) > g
      1. H0 fails for t > \(X_{1 - \alpha}\)
    3. H0: \(\sigma^{2}\) = g, Ha: \(\sigma^{2}\) / g
      1. H0 fails for t < \(X_{1 - \frac{a}{2}}\)
      2. H0 fails for t > \(X_{1 - \frac{a}{2}}\)
  • Confidence Interval

    If the original population of data that a sample is taken from is normally distributed then the expression\(\frac{\left( n - 1 \right)s^{2}}{\sigma^{2}}\) has a chi square distribution with \(n - 1\) degrees of freedom.

    Thus it can be shown:

    For a confidence level of \(1 - \alpha\)

    \[\frac{\left( n - 1 \right)s^{2}}{\chi_{\frac{\alpha}{2},\ \ d.f.}}\ \ \leq \ \ \sigma^{2}\ \ \leq \ \ \frac{\left( n - 1 \right)s^{2}}{\chi_{1 - \frac{\alpha}{2},\ \ d.f.}}\text{\ \ }\]

    \[\sqrt{\frac{\left( n - 1 \right)s^{2}}{\chi_{\frac{\alpha}{2},\ \ d.f.}}\text{\ \ }}\ \leq \ \ \sigma\ \ \leq \ \ \sqrt{\frac{\left( n - 1 \right)s^{2}}{\chi_{1 - \frac{\alpha}{2},\ \ d.f.}}}\text{\ \ }\]

Chi Distribution

Used for analysing categorical data, each cell should contain at least 5 counts.

This is found by using a computer to measure the distribution of categorical data in samples.

Hypotheses

H0: Cells match hypothesised value for distribution at α confidence using the sample to predict the population

Ha: Cells do not match the hypothesised value

Test Statistic (p. 2 Wk. 13 Lecture Notes)

\[x^{2} = \sum_{i = 1}^{k}{\left( \frac{\left( e_{i} - o_{i} \right)^{2}}{e_{i}} \right)\ \ \sim\ \ \ Chi\ Distribution}\]

Where:

  • ei is the expected value
    • ei is equal to the legitimate expected value (which is usually the average value)
    • OR, ei = $\frac{\left( \text{row\ total} \right)\left( \text{Column\ total} \right)}{(Grand\ Total)}\ $
  • oi is the observed value

Rejection Region

H0 fails for x2 > x2, n-1, α

α: Is the probability that H0 is rejected incorrectly

The larger x2, n-1, α, the smaller α, the harder it is to accept H0.

n: Is the number of cells and d.f. is the degrees of freedom = n-1

Calculating Power

Statistical Power is the Probability that an incorrect null hypothesis will be rejected, it is best shown by way of example.

Actual Situation Decision  
  Do Not Reject H0 Reject H0
H0 is True Correct Decision(1-\(\alpha\)) Type 1 Error (\(\alpha\))
H0 is False Type 2 Error(\(\beta\)) Correct Decision (\(1 - \beta\))

Example (1.1 From Week 1 Lecture Notes)

An ISP stated that users average 10 hours a week of internet usage, it is already known that the standard deviation of this population is 5.2 hours. A sample of n=100 was taken to verify this claim.

A worldwide census determined that the average is in fact 12 hours a week not 10 hours.

* Data \[H_{0}:\text{Users\ Average\ ten\ hours\ a\ week\ of\ internet\ usage}\]
:PROPERTIES:  
:CUSTOMID: data-1 \[H_{a}:\text{Users\ Average\ over\ ten\ hours\ a\ week\ of\ internet\ usage}\]
:END:  
   
\[n = 100\]  
   
\[\sigma = 5.2\]  
   
\[\mu = 10\]  
   
\[\overline{x} = 11\]  
   
\[\mu_{\text{true}} = 12\]  
   
\[\alpha = 0.05\]  
   
\(\beta\) = ???  

Step 1: Find the Critical Sample Mean

(\({\overline{x}}_{\text{Critical}})\)

:CUSTOMID: step-1-find-the-critical-sample-mean-overlinextextcritical

Find the Critical Value of \(\overline{x}\) (the sample mean) that determines when H0 would be rejected

\[Z = \frac{\overline{x} - \mu}{(\frac{\sigma}{\sqrt{n}})}\]

\[\overline{x} = \mu + z\ (\frac{\sigma}{\sqrt{n}})\]

For \(\alpha\) = 0.05 H0 is rejected for Z> Z0.05 which translates to Z > 1.645, thus substitute z for 1.645 and solve for the sample mean (\(\overline{x}\)

\[\overline{x} = \left( 10 \right) + \left( 1.645 \right)\left( \frac{\left( 5.2 \right)}{\sqrt{\left( 100 \right)}} \right) = \ 10.8554\]

Thus H0 Is rejected for a sample mean of 10.8554 hours per week at a confidence level of \(\alpha\) = 0.005.

Step 2: Find the Difference between the Critical Sample Mean and the

True Mean as a z-value.

:CUSTOMID: step-2-find-the-difference-between-the-critical-sample-mean-and-the-true-mean-as-a-z-value.

Find the difference between the true population mean and the sample mean that would reject the hypotheses, then find that small section of the distribution relative to a standard normal distribution.

\[Z = \frac{{\overline{x}}_{\text{Critical}} - \mu_{\text{True}}}{(\frac{\sigma}{\sqrt{n}})}\]

\[Z = \frac{10.8554 - 12}{(\frac{5.2}{10})} = \ - 2.20115\]

Step 3: Find \(\beta\)

\[\beta = P\left( Type\ 2\ Error \right)\]

\[\beta = P\left( H_{0}\text{\ is\ not\ rejected\ } \right|H_{0}\ Is\ wrong)\]

\[\ \beta = P\left( \overline{X} > {\overline{x}}_{\text{Critical}}\ \right|\mu = 12)\]

\[= P\left( Z > - 2.20115 \right)\]

\[= 0.0136\]

\[\therefore The\ probability\ of\ a\ type\ 2\ error\ in\ this\ test\ is\ 1.36\%\]

Step 4: Find the Power (1-\(\beta)\)

\[Power = \left( H_{0}\text{\ Is\ rejected\ } \right|\ H_{0\ }is\ false)\]

\[= P\left( \overline{X} < {\overline{x}}_{\text{Critical}}\ \right|\mu = 12)\]

\[= 1 - \beta\]

\[= 1 - 0.0136\]

\[= 0.9864 = 98.6\%\ \]

Hypothesis Testing Two Population Samples

Topic 1B | Week 2 | Material Drawn From Week 2 Lecture

When comparing two population means t-distributions and standard normal distributions can be used, when comparing multiple population means an ANOVA table must be used (Topic 3 of this Unit).

Difference between Two Population Means (Independent Populations)

Hypotheses3

The hypothesis in all scenarios of comparing independent population means is the same.

\(H_{0}:\ \mu_{1} - \mu_{2} = D\) (D can be anything e.g. zero) 1. \(H_{a}:\mu_{1} - \mu_{2} < D,\ \ or\)
   
  2. \(H_{a}:\mu_{1} - \mu_{2} > D,\ \ or\)
   
  3. \(H_{a}:\mu_{1} - \mu_{2} \neq D\)

Where \(\sigma\) is known

If the population Variance is known the Standard Normal Distribution is used.

  • Where Variance between populations are Equal

    If the population variance is equal it doesn't matter, it is the same equation as if they were different, just sub in identical values for \(\sigma\). (Refer below)

  • Where Variance between populations are NOT Equal
    • Hypotheses

      As Above

    • Test Statistic

      The test statistic used is essentially a combination of both test statistics given by:

      \[Z = \frac{(\overline{x_{1}} - \overline{x_{2}})\ - (\mu_{1} - \mu_{2})}{\sqrt{\frac{\sigma_{1}^{2}}{n_{1}} + \frac{\sigma_{2}^{2}}{n_{2}}}}\ ,\ where\ \left( \mu_{1} - \ \mu_{2} \right) = D\]

      If the test was of a single population mean it would simplify back to:

      \[Z = \frac{\left( {\overline{x}}_{1} - \mu_{1} \right)}{\sigma/\sqrt{n}}\]

    • Rejection Region

      H0 is rejected if:

      1. \(H_{a}:\mu_{1} - \mu_{2} < D\)
        1. \(Z < {- Z}_{\alpha}\)
      2. \(H_{a}:\mu_{1} - \mu_{2} > D\)
        1. \(Z > Z\alpha\)
      3. \(H_{a}:\mu_{1} - \mu_{2} \neq D\)
        1. \(Z < - Z_{\frac{\alpha}{2}}\ or\ Z > Z_{\alpha}\text{\ Which\ is\ the\ Same\ as\ }\left| Z \right| > |Z_{\frac{\alpha}{2}}|\)
    • Confidence Interval

      A ($1 - α)$% Confidence interval of \((\mu_{1} - \mu_{2})\) is given by

      \[\left( \overline{x_{1}} - \overline{x_{2}} \right) \pm Z_{\frac{\alpha}{2}} \times \sqrt{\frac{\sigma_{1}^{2}}{n_{1}} + \frac{\sigma_{2}^{2}}{n_{2}}}\]

Where \(\sigma\) is unknown and \(s\) is used

If the population variance is not known the Student's t Statistic is used4 because the sample variance is being used as an estimate and the t-distribution compensates for that.

  • Small Sample Sizes5

    If sample sizes are considered small then the degrees of freedom does not equal (n1-1) + (n­2-1), instead Welch's Approximation must be used.

    \[d.f.\ = \left( n_{1} - 1 \right) + \left( n_{2} - 1 \right)\ ,\ for\ n\ \geq 30\]

    \[d.f.\ = \frac{\left\lbrack \frac{s_{1}^{2}}{n_{1}} + \frac{s_{2}^{2}}{n_{2}} \right\rbrack^{2}}{\frac{{(\frac{s_{1}^{2}}{n_{1}})}^{2}}{n_{1} - 1\ } + \frac{{(\frac{s_{2}^{2}}{n_{2}})}^{2}}{n_{2} - 1\ }},\ for\ n\ < 30\]

    It should be noted that d.f.Welch < \(\left( n_{1} - 1 \right) + \left( n_{2} - 1 \right)\)

  • Where Variance between populations are NOT Equal6

    Unless there is evidence to the contrary variances are usually considered to be equal, this method is sensitive to sample sizes due to the effect they have on degrees of freedom.

    • Hypothesis

      As above

    • Test statistic

      The test statistic is essentially a combination of both the statistics, the t-distribution is used because \(\sigma\) is unknown.

      \[t = \frac{(\overline{x_{1}} - \overline{x_{2}})\ - (\mu_{1} - \mu_{2})}{\sqrt{\frac{s_{1}^{2}}{n_{1}} + \frac{s_{2}^{2}}{n_{2}}}}\ ,\ where\ \left( \mu_{1} - \ \mu_{2} \right) = D\]

    • Rejection Region

      H0 is rejected if:

      Where degrees of freedom (d.f.) is given by (d.f. = n1 + /n2 –

      1. (for n1 > 30 and n2 > 30)/
      2. \(H_{a}:\mu_{1} - \mu_{2} < D\)
        1. \(t < {- t}_{\alpha,\ d.f.}\)
      3. \(H_{a}:\mu_{1} - \mu_{2} > D\)
        1. \(t > t_{\alpha,\ d.f.\ }\)
      4. \(H_{a}:\mu_{1} - \mu_{2} \neq D\)
        1. \(t < - t\ or\ t > t_{\alpha,\ d.f.}\text{\ Which\ is\ the\ Same\ as\ }\left| t \right| > |t_{\frac{\alpha}{2},\ d.f.}|\)
    • Confidence Interval

      \[\left( \overline{x_{1}} - \overline{x_{2}} \right) \pm t_{\frac{\alpha}{2},\ d.f.} \times \sqrt{\frac{s_{1}^{2}}{n_{1}} + \frac{s_{2}^{2}}{n_{2}}}\]

  • Where Variance between populations are Equal (N needn't be > 30)

    For this method sample sizes needn't necessarily be large.

    • Hypothesis

      As above

    • Test Statistic

      The test statistic essentially averages the variances to get a more accurate result, the average variance is referred to as the pooled variance (\(s_{p}^{2}\))

      \[t = \frac{(\overline{x_{1}} - \overline{x_{2}})\ - (\mu_{1} - \mu_{2})}{s_{p}\sqrt{\frac{1}{n_{1}} + \frac{1}{n_{2}}}}\ ,\ where\ \left( \mu_{1} - \ \mu_{2} \right) = D\]

      Where pooled Variance is given by:

      \[s_{p}^{2} = \frac{s_{1}^{2}\left( n_{1} - 1 \right) + S_{2}^{2}(n_{2} - 1)}{n_{1} + n_{2} - 2\ }\]

    • Rejection Region

      H0 is rejected if:

      Where degrees of freedom (d.f.) is given by (d.f. = n1 + /n2 –

      1. (for n1+n­2 > 30)/
      2. \(H_{a}:\mu_{1} - \mu_{2} < D\)
        1. \(t < {- t}_{\alpha,\ d.f.}\)
      3. \(H_{a}:\mu_{1} - \mu_{2} > D\)
        1. \(t > t_{\alpha,\ d.f.\ }\)
      4. \(H_{a}:\mu_{1} - \mu_{2} \neq D\)
        1. \(t < - t\ or\ t > t_{\alpha,\ d.f.}\text{\ Which\ is\ the\ Same\ as\ }\left| t \right| > |t_{\frac{\alpha}{2},\ d.f.}|\)
    • Confidence Interval

      A \(\left( 1 - \alpha \right)\%\) Confidence Interval of \({(\mu}_{1} - \ \mu_{2})\) is given by:

      \[\left( \overline{x_{1}} - \overline{x_{2}} \right) \pm t_{\frac{\alpha}{2},\ d.f.} \times s_{p}\sqrt{\frac{s_{1}^{2}}{n_{1}} + \frac{s_{2}^{2}}{n_{2}}}\]

Difference between Two Population Means (Dependent Populations or

Paired Samples)

Goodness-of-fit Tests

A goodness-of-fit test provides a means by which one can compare how data behaves to how one thinks it behaves.

If it can be safely assumed that data in a population follows a normal distribution, the chi-squared statistic is a tool that can be used for this.

Normal Samples

If a sample is randomly taken from a normally distributed population then:

\[Z_{i} = \frac{\overline{x_{i}} - \mu}{\sigma/\sqrt{n}}\ \ \sim\ Standard\ Normal\ Distribution\]

\[Z_{i} = \frac{\ o_{i} - e_{i}}{\sqrt{e_{i}}}\]

Proof of Expected Frequency12

The Probability of two independent events, A and B, can be given by:

\[P\left( \text{A\ }\text{and}\text{\ B} \right) = P\left( A \right) \times P(B)\]

So if data held in rows and columns is independent (as the null hypothesis suggests) then the probability of the row event happening and the column event happening is given by:

\[P\left( \text{Row\ event\ }\text{and\ }\text{Column\ Event} \right) = P\left( \text{Row\ Event} \right) \times P(Column\ Event)\]

The expected frequency of an event is the number of trials \(n\) multiplied by the probability of an event.

For example if a die were to be rolled 6 times it would be expected that the number 4 would appear twice.

\[n\left( \text{Row\ event\ }\text{and\ }\text{Column\ Event} \right) = P\left( \text{Row\ event} \right) \times P\left( \text{Column\ Event} \right) \times n\]

\[n\left( \text{Row\ event\ }\text{and\ }\text{Column\ Event} \right) = \frac{n \times P\left( \text{Row\ event} \right) \times P\left( \text{Column\ Event} \right) \times n}{n}\]

\[n\left( \text{Row\ event\ }\text{and\ }\text{Column\ Event} \right) = \frac{No.\ of\ Row\ events \times No.\ of\ Column\ Events}{n}\]

Thus the Expected frequency is given by:

\[expected\ frequency = \ n\left( \text{Row\ event\ }\text{and\ }\text{Column\ Event} \right) = \frac{Row\ Events \times Column\ Events}{n}\]

Analysis of Variance

When more than two independent population means are compared a t-distribution will no longer cut it, instead a technique known as Analysis of Variance must be used.

This compares different causes of variation between groups and provides a test statistic to determine whether such differentiation is significant.

This is a comparison of the variance caused by random error and the variance caused by differing treatments being tested in order to yield some result.

In essence there is only on block of treatments however a two-way or greater might have many blocks of treatments, that is to say, a comparison of the results yielded by different treatments under a variety of circumstances.

For an ANOVA table to be used observations between treatments must:

Two-Way ANOVA

Block Design

One-way Anova deals with only one source of explained variation, treatment to treatment variation, whereas Two-Way Anova deals with different sources of variation, such as the distribution of fish in different streams at different depths for each respective stream.

For a two-way ANOVA the total variation is partitioned in to three sources:

  1. Between Treatments (A)
    1. E.g. Fish numbers between streams
  2. Between Blocks (B)
    1. E.g. Fish numbers at different depths of the respective streams
  3. Within Treatments, i.e. random error (E)
    1. The variation of the data taken at said streams at said depth

Sum of Square Deviants

\[\text{SS}A_{\text{Factor\ A\ \ or\ Treatment}}\]

This is the variation of the data between all the treatments of data (e.g. the streams)

\[SSA = \ \sum_{i = 1}^{r}\left\lbrack n_{i}\left( \overline{x_{i}} - \overline{\overline{x}} \right)^{2} \right\rbrack\]

\[\text{SS}B_{\text{Factor\ B\ or\ Block}}\]

This is the variation of the data between the blocks of said treatments (e.g. the depths)

\[\text{SS}E_{\text{Error}}\]

The variation caused by error is the amount of variation that occurs by way of random errors within all the treatments throughout the various blocks and throughout the various treatments.

\[SSE = SST - SSA - SSB\]

Tukey's test is essentially a t-test except that it corrects for experiment-wise error rate, as multiple comparisons are bing made the probability of making a type 1 error increases – Tukey's test corrects for that

Tukey's Test assumes that the data under scrutiny is:

  2. The means are from Normally Distributed Populations
  3. There is equal variation across observations

A regression analysis implies that there exists some linear relationship between the variables \(x\) and \(y\).

A way to measure this is by using the value:

\[r = \frac{SS_{\text{xy}}}{\sqrt{SS_{x} \times SS_{y}}}\]

\[b_{1} = \frac{SS_{\text{xy}}}{SS_{x}} = r \times \ \sqrt{\frac{SS_{y}}{SS_{x}}}\]

This measures the linear association between the two variables for \(\left| r \right| < 1\).

If the value of \(r\) is negative 1 there is a perfect negative linear relationship between the variables and likewise for positive values of \(r\).

Smoothing Techniques

Where data is erratic fitting a trend line could just be a waste of time, another approach is to just smooth the data out by running an average through it.

Trailing Moving Average (TMA)

This averages the data over the last \(m\) periods:

\[{\widehat{y}}_{t} = \frac{y_{t} + t_{t - 1} + \ldots + y_{t - m + 1}}{m}\]

Choosing more periods i.e. a larger \(m\) yields a ‘smoother' average but requires more data.

The value of \(\widehat{y_{t}}\) can be usesd as a forecast for period \(+ 1\) , that is it is a noe-period-ahead forecast.

Centred Moving Average (CMA)

This method looks forward and backward in time to express the current ‘forecast' as a mean of the current observation and observations on both sides of the current data.

For example, where \(m = 3\) periods are used, the CM is:

\[{\widehat{y}}_{t} = \frac{y_{t - 1} + y_{t} + y_{t + 1}}{3}\]

When m is odd like 3, the CMA is easy to calculate, but when it is even, the mean of an even number of data points would lie between two data points and would be incorrectly centred, in this case a double moving average is taken to get the resulting CMA centred properly.

Exponential Smoothing

The exponential smoothing model is a special kind of moving average.

It is used where data has up=and=down movements with no consistent trend.

The updating formula is:

\[F_{t + 1\ } = \alpha y_{t} + \left( 1 - \alpha \right)F_{t}\]

Where:

  • \(F_{t + 1} =\) the forecast for the next period
  • \(\alpha\) = the ‘smoothing' constant, 0 < \(\alpha\) < 1
  • \(y_{t} =\) the actual data value in period \(t\)
  • $Ft = \ $the previous forecast for period \(t\)

The next forecast (\(F_{t + 1})\) is a weighted average of the current data (\(y_{t}\)) and the previous forecast (\(F_{t}\))

The value of \(\alpha\) (the smoothing constant) is the weight given to the latest data, where a low \(\alpha\) value would give little weight to the most recent observation, a larger \(\alpha\) value means the forecast quickly adapts to recent data.

\(\alpha = 1\) means no smoothing at all, (i.e. the forecast value for the nesxt period is the same as the latest data point.)

It should be noted that \(F_{t - 1}\) is always dependent on the value of \(F_{t}\)

  • Initialising the Process

    Because the forecast values all run off of one-another it is necessary to pick an initial forecast value, there are two ways to go about this

    • Method A – No Forecast

      Just set the first forecast as the first data value:

      \[F_{1} = y_{1}\]

      An unusual \(y_{1}\) value may mean it could take a few iterations before the forecasts stabilize.

    • Method B – Average Value

      Average the first 6 data values.

      Set \(F_{1} = \frac{1}{6}\left( y_{1} + y_{2} + \ldots + y_{6} \right)\)

      This however consumes a bunch of data and is still somewhat vulnerable to an unusual \(y\) value.

Measuring the utility of the Regression

Slope test

Given the line \(y = \alpha + \beta x\) we can use \(b\) as a sample value for the actual value of \(\beta\).

If \(\beta = 0\) then there must be no linear correlation between the variables.

If \(\beta = 0\) is refuted at some confidence level, then there is a \(\beta\) value such that a linear correlation may fit.

The point of this test is that a linear correlation may definitely exist, but there is a shit tonne of random error, so a linear correlation may exist at a high significance level but have a really low coefficient of determination.

Regression Diagnostics

A residual is the error from an observed point to the predicted point, so if \(e\) is the error or residual, \(y\ \hat{}\) is the fitted point (i.e. the line) and \(y\) is the observed point:

\[e = y - \widehat{y}\]

Usually \(i\) denotes the number of the observation:

\[e_{i} = y_{i} - \widehat{y_{i}}\]

In the lecture notes the standardised residual was used, this value is:23

\[std(e_{i}) = \frac{e_{i}}{s_{e}}\]

Where:

\(std(e_{i})\) Is the standard residual

\(e_{i}\) Is the residual for the \(i^{\text{th}}\) data point

\(s_{e}\) Is the standard deviation of the residual.

What we are testing

An assumption made when forming a linear regression is that the error (\(\varepsilon\)) in the model (\(y = \beta_{1} \bullet x + \beta_{0} + \varepsilon\)) is normally distributed (and hence has a constant variation).

If this assumption can be rejected, then the linear regression may not be an appropriate model:24

Individually Creating the Residual Diagnostics

The actual method to create the diagnostics is:

Residuals (i.e. the random error) should follow a normal distribution with a mean value of 0, this assumption can be tested by various plots:

  1. Plotting residuals (\(e\)) against fitted values (\(y\))
    1. There should be no pattern
  2. Plotting the square root of the absolute value of residuals \(\sqrt{|e|}\) against the fitted values (\(y\))
    1. Same as above but tests more so the assumption of constant variance of the residual
  3. Normal Q-Q plot (residuals plotted against fitted values)
    1. Tests the assumption that the errors come from a normal distribution
  4. Plotting residuals against Cook's Distance
    1. Cook's distance is a measure of how much a data point affects the slope of the regression, it is the leverage of that data point.

These plots are such that the residual is always put along the $y$-axis.

In simple linear Regression the significance of the regression could be determined by testing whether the slope \(\text{β\ }\) was more than 0 via a t-statistic, the t-statistic can only compare two values and for Simple Linear Regression is equivalent to an F-statistic.

In Multiple Linear Regression the t-statistic is not equal to the F-statistic, and to determine whether atleast one of the slopes is significant an Analysis of Variation (ANOVA) testing strategy must be employed to get an \(F - Statistic\).

A hypothesis test of significance would be:

\[H_{0}:\beta_{1} = \beta_{2} = \beta_{3}\ldots\beta_{k} = 0\]

Model Performance Trade Offs

Rare Events

Rare events create challenges for classification models, where an event is rare (e.g. donation), predicting the opposite (refraining from donating) can result in a very high accuracy, as happened in our R script with the donors data set.

ROC and AUC curves

ROC curves are an important tool for comparing models and selecting the best model for project needs.

When used with a single model it can help to visualise the trade off between True positives and False positives for the outcome of interest.

Discrete or Categorical Data

Where data is categorical

e.g.

  1. poor
  2. Middle Class
  3. Rich

It is necessary to convert that data to a factor before it is used in a logistic regression.

Even where the data may be numeric as it is in the above example, it may be appropriate to convert it to a factor so each variable is considered on its own.

Remember when creating the factor to specify the levels of the factor if it is ordered:

factor(donors$wealthrating, levels = 0:3, labels = c(0,1,2,3)

Interaction Effects

Where data may interact with each other (e.g. stimulants and depressants neutralising perceived mood changes or smoking in combination with obesity raising the risk of heart disease more sharply than if they were separate), R will perform a logistic regression for the interaction effect between an x1 and x2 variable with the following formula:

glm(y ~ x1 * x2, family = “binomial”)

Automatic Feature Selection

Unlike some machine learning methods, regression requires the human to select the parameters before the model is made.

So in the case of donations, it was possible to use insight into the potential factors that may be predictive of whether or not a person would make a donation.

If we don't have that insight ahead of time or there are too many predictors to go through individually a process called automatic feature selection can be used.

Stepwise regression

Involves building a regression model step by step evaluating each predictor to determine which ones add value to the model.

The potential caveats of stepwise regression include:

  • Forward and Backward stepwise selection may reach different models
  • Neither method is guaranteed to find the best possible model
  • There are concerns that a stepwise model violates some of the principles that allow a regression model to explain data as predict.
    • This might not be a large concern if you are ONLY interested in the predictive power of the regression model.
      • The use of stepwise does not mean that the models predictions are worthless, it simply means that the model may overstate or understate the importance of some of the predictors
  • Feature selection methods like stepwise regression allow the model to be built in the absence of theory or even common sense, this can result in a model that is counterintuitive in the real world

Stepwise regression should be considered one tool for exploring potential models in the absence of some other good starting point

Though stepwise regression is frowned upon, it may still be useful for building predictive models, it would NOT be correct to say that a stepwise model's predictions could not be trusted.

  • Backward Deletion

    Backward stepwise deletion begins with a model that contains all of the predictors

    It then checks to see what happens when each one of the predictors is removed from the model.

    If removing a predictor does not substantially affect the models ability to predict the outcome then it can be safely deleted.

  • Forward Selection

    This is like backward deletion but applied in the other direction

    Start with a model that has no predictors, examine each predictor if any has any offers any improvement to the models predictive power.

    Predictors are added step by step until no new predictors add substantial value to the model.

    Backward and forward stepwise selection may end up with completely different models.

The Runs test is for testing the randomness of a sample.
