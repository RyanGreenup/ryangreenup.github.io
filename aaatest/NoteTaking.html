Advantages to org-mode

  • The JS it adds onto the HTML is phenomenal
  • the mathjax is really powerful, I love that I can layout proofs by using tables like I did for proof of Fermats Little Theorem
  • Actually I also love how powerful the interlinking is, it is again phenomenal.

Yeah, my notes will definitely be a combination of org, Boostnote, RawMD / Vnote

  • The problem with org mode is that if some of the math is broken it takes the entire document with it, which means no export and no PDF
    • whereas is markdown you only lose that one block
    • this can be overcome in previewing LaTeX fragments and or exporting only headlines
    • in beorg your fucked, if there is an error in the math above, it will carry on down to later sections sabotaging the whole document
    • the other problem is that the editor in beorg is absolutelyfucking useless.
      • and org mode might render the math where beorg doesn't, but that's more of a problem with pandoc.
    • If you were going to do one big note, corresponding to an entire class in org-mode you would have to accept that every step of the way it will be necessary to test for compatability with beorg
    • beorg Does not support inline math.

Basically the biggest issue with org mode is that the beorg app sucks.

Moreover `org-mode` has such a robust syntax that it's fairly easy to sync `org-mode` with a corresponding markdown file meaning.

So basically `org-mode` and boostnote are not going to be able to be integrated with iOS

File Names

Because locate and helm-locate search specifically filenames, filenames should be:

  1. a unique name
  2. a descriptive name

that way you can save time be just searching the file name. Although you can search the title of the document and terms contained in there, in doing so you are sacrificing a benefit in that you pipe results of locate into results of ag and locate will always be faster over an entire system.

constricting ag to just the notes folder would likely fix this that's something that will have to be implemented in bash or helm, I can't think of any reason not to use descriptive, unique file names (that are syntactically correct of course).
